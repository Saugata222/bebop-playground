Get Started with Bebop Shell

Set up the project locally, give the AI full context, and create your own design experiments on top of the Copilot Shell prototype.

Setup
1
Install prerequisites Download and install these three things: • Node.js (v18 or later) — runs the build scripts • VS Code — the code editor • GitHub Copilot extension for VS Code — the AI assistant that will write code for you
2
Get the project If you already have a bebop-design-system.zip or bebop-playground folder from a previous download, you can use that — just make sure it's the latest version. Otherwise, download or clone:
Download the zip and extract it into a folder: Download from OneDrive
git clone https://github.com/Saugata222/bebop-playground.git
3
Open in VS Code and install dependencies Open the extracted folder in VS Code (File → Open Folder), then open the terminal (Ctrl+` or Cmd+`) and run:
npm install
This installs TypeScript and Fluent UI components. Takes about 30 seconds.
4
Verify it works Build all previews and start the local server:
npm run dev
Open localhost:8765/preview/dist/shell.html in your browser. You should see the interactive Copilot shell.
Giving AI full context
5
Give the AI full project context The project includes documentation files that tell the AI everything about the architecture, conventions, and how to build. Start every new chat session by asking the AI to read them:
"Read the CLAUDE.md file and the .github/copilot-instructions.md file. Understand the full project architecture before we start."
This is the most important step. These files contain:
CLAUDE.md — Complete architecture guide: file structure, token system, CSS variable pipeline, how preview generators work, every CSS class state and DOM element.
.github/copilot-instructions.md — Condensed version of the same info. Copilot may auto-read this, but it's good to be explicit.
Once the AI confirms it has read them, it will know every file, every convention, and how to build. You won't need to explain the project yourself.
6
Orient yourself Before making changes, ask the AI to explain what you're looking at. Try these prompts:
"Explain the file structure of this project. What does each folder do?"
"Walk me through preview/src/shell.ts. What are the key sections and how does it generate the HTML?"
"What are the existing flow explorations? What does seamlessConnect.ts build vs connectorsGoldenFlow.ts?"
7
Reference Figma designs in your prompts If you have a Figma design to implement, paste the Figma link directly into Copilot Chat. The AI can read Figma nodes and extract colors, spacing, and layout:
"Here's my Figma design: [paste Figma URL]. Implement this as a new feature in the shell."
No need to manually redline values — the AI extracts them from Figma directly.
Creating your own experiments
8
Option A — Modify the shell directly The simplest approach. Ask the AI to add or change things in the existing shell. The AI edits preview/src/shell.ts and rebuilds:
"Add a citations panel below the response text in the shell, showing 3 source cards with app icons"
"Add a persistent banner below the input that shows 'Copilot can access M365, Jira, GitHub +3 more'"
"When I type a message containing 'tickets', show an inline suggestion to connect Jira"
9
Option B — Create a new flow exploration If you want a standalone prototype without modifying the main shell, ask the AI to create a new flow file. This copies the shell as a starting point and layers your experiment on top:
"Create a new flow exploration called 'myExperiment' based on the shell. It should start with [describe your flow]."
This creates preview/src/myExperiment.ts → builds to preview/dist/myExperiment.html. The main shell stays untouched. See existing examples: seamlessConnect.ts (FRE + onboarding flow) and connectorsGoldenFlow.ts (connector setup flow).
10
Build, preview, iterate After any change, rebuild and check the result:
npx tsx preview/src/shell.ts
Then refresh the browser. Replace shell.ts with your file name if working on a custom flow.
"Build the shell preview and give me a preview"
Iterate fast: describe what's wrong or what to change next, and the AI will edit and rebuild. You don't need to touch the code yourself.
Tips & tricks
Be specific about what you want Instead of "add a banner", say "add a horizontal banner below the chat input, 40px tall, #f5f5f5 background, showing app icons for M365, Jira, GitHub with a '+3 more' link on the right".
Reference existing components "Make it look like the add menu overlay" or "use the same source card style from seamlessConnect" — the AI knows every component.
Check the browser console If something looks broken, open browser DevTools (F12) → Console. JS errors in preview generators fail silently. Paste any errors back into the chat.
Available icons 64 Fluent icons + app logos in src/components/icons/. Ask the AI: "What icons are available?" and it will list them all.
Undo mistakes VS Code has unlimited undo (Cmd+Z). Or tell the AI: "Revert that last change".
Quick reference
npm run build — rebuild shell only
npm run build:all — rebuild all previews
npm run serve — start server on port 8765
npm run dev — build + start server
npx tsc --noEmit — type-check
npx tsx preview/src/shell.ts — rebuild single file
