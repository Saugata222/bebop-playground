# Bebop Design System

TypeScript design token library for the Bebop Design System. Tokens are hand-authored TS objects that flatten to CSS custom properties. Components will be implemented with `@fluentui/web-components` (FAST Element).

## Quick Start

```sh
npm install
npm run typecheck        # Type-check (no build output)
npm run preview:build    # Generate all HTML previews
npm run preview:serve    # Serve at http://localhost:8765
```

## Architecture

```
src/
  tokens/                  Foundation tokens + contract utilities
    foundations/            color, typography, spacing, radius, stroke, elevation, grid, state
    types.ts               CSSVarMap, TokenValue, TokenLayer, TOKEN_PREFIX
    contract.ts            flattenTokens(), tokenVar(), rootCSS
    index.ts               Re-exports foundations + contract

  components/              Component design tokens
    shared/                Reusable tokens (focusRing, disabledState, selectedAffordance)
    primitives/            Atomic components (button, tag, divider, tab, tabList, toggle, scrollbar, suggestionChips)
    compound/              Composed components (toolbar, chatInput, canvasChat, dialog, menu, nav, header, addMenu, sourcesMenu)
    icons/                 Fluent UI System Icons (SVG + PNG + GIF assets)
    index.ts               Barrel export

  patterns/                Page-level compositions (future)
  utils/                   Shared helpers (future)

preview/
  src/                     TypeScript preview generators
    _icons.ts              Shared SVG icon constants (imported by generators)
  dist/                    Generated HTML files (gitignored)
```

## Token Layers

| Layer        | CSS prefix | Location                |
|-------------|-----------|-------------------------|
| Foundations  | `--f-`    | `src/tokens/foundations/` |
| Components  | `--c-`    | `src/components/`        |
| Patterns    | `--p-`    | `src/patterns/`          |

## Code Style

- `as const` on every exported object/array
- Named exports only — no default exports
- `import type` for type-only imports
- camelCase for all identifiers and object keys
- No enums — use `as const` objects
- JSDoc file headers on every file

## Preview System

Each component has a standalone HTML preview generated by a TypeScript file in `preview/src/`. The shell preview (`preview/dist/shell.html`) composes all components into the full Copilot interface.

```sh
npx tsx preview/src/shell.ts    # Build shell preview
npx tsx preview/src/button.ts   # Build button preview
```

Serve from project root: `python3 -m http.server 8765`  
View at: `http://localhost:8765/preview/dist/shell.html`
